# Hebrew i18n + RTL Implementation Plan

> Execute task-by-task. After EVERY task: `pnpm test && pnpm typecheck && pnpm lint && pnpm build` must be green, and every `t()` key must exist in both `messages/en.json` and `messages/he.json`.

**Goal:** Whole UI in Hebrew (RTL) + English, auto-detected from the browser with a persisted toggle; AI outputs stay English. Library: next-intl (App Router, cookie-based, no locale in URL).

---

## Task A — Infrastructure (i18n core + toggle + sidebars)

**Files:** `package.json` (add next-intl), `next.config.ts`, `src/i18n/locale.ts` (+ `locale.test.ts`), `src/i18n/request.ts`, `messages/en.json`, `messages/he.json`, `src/app/layout.tsx`, `src/lib/actions/locale.ts`, `src/components/LanguageToggle.tsx`, `src/app/(app)/layout.tsx` + `src/app/admin/layout.tsx` (toggle + nav via `common`/`nav` namespaces).

1. `pnpm add next-intl`.
2. TDD `src/i18n/locale.ts`: `SUPPORTED_LOCALES=['en','he']`, `DEFAULT_LOCALE='en'`, `resolveLocale(cookie, acceptLanguage)` — cookie `he`/`en` wins; else `he` if Accept-Language prefers Hebrew; else `en`.
3. `src/i18n/request.ts` — `getRequestConfig` resolving locale (cookie + Accept-Language) and importing `messages/<locale>.json`.
4. `next.config.ts` — wrap with `createNextIntlPlugin('./src/i18n/request.ts')`.
5. `src/app/layout.tsx` — `getLocale()`/`getMessages()`, `<html lang dir>`, `NextIntlClientProvider`.
6. `src/lib/actions/locale.ts` — `setLocale` server action (cookie + `revalidatePath('/','layout')`).
7. `src/components/LanguageToggle.tsx` — client toggle (he↔en).
8. `messages/{en,he}.json` — seed `common` + `nav` namespaces.
9. App + admin sidebars: use `nav` keys, add `<LanguageToggle/>`; convert directional classes (`border-r`→`border-e`).
10. Verify; commit.

## Task B — Auth + onboarding (`auth`, `onboarding` namespaces)
Translate `(auth)/login`, `(auth)/signup`, and the onboarding wizard + steps. Add keys to both files. Convert directional classes. Verify; commit.

## Task C — App (`offers`, `campaigns`, `billing` namespaces)
Translate offers (list, new, detail tabs Overview/Scorecard/Verdict/TestKit/Compliance chrome — labels/headings/buttons only, NOT AI payloads), campaigns (list, results form, diagnosis chrome), billing. Convert directional classes (`text-left`/`pl-5`→logical). Verify; commit.

## Task D — Admin (`admin` namespace)
Translate `/admin/*` page chrome (headings, table headers, buttons, empty states) across ai-runs, prompts, eval, discovery, kill-switches, compliance, invite-codes, failed. AI payloads + data stay as-is. Verify; commit.

## Final
Full suite + build green; manual toggle check (language + dir flip, persists); merge to main; deploy is automatic (Vercel) — no edge/migration. Update CLAUDE.md.

## Out of scope
AI output translation; URL locale; languages beyond he/en.
